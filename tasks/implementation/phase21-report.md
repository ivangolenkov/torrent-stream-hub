# Phase 21: Report (Fix Pause State Override by Stream Manager)

## 1. Что было сделано
* Проанализирована проблема, при которой торрент, находящийся на паузе, продолжал или начинал скачиваться после завершения стриминга файла.
* Установлено, что метод `setSequentialMode` в `internal/engine/stream.go` при отключении "эгоистичного режима" (когда `enable=false`) слепо выставлял всем файлам приоритет `PiecePriorityNormal`, что заставляло движок `anacrolix/torrent` проигнорировать предыдущую команду "Пауза".
* В `setSequentialMode` добавлена безопасная проверка состояния торрента `mt.state`.
* Если стрим отключается, а торрент при этом находится в одном из неактивных состояний (`StatePaused`, `StateError`, `StateDiskFull`, `StateMissingFiles`), приоритет файлов устанавливается в `PiecePriorityNone`, а также вызывается метод `mt.t.CancelPieces(0, mt.t.NumPieces())`, чтобы гарантировать, что загрузка не возобновится.
* Запущены тесты (`go test ./...`) — все проверки прошли успешно.
* Выполнена пересборка клиентской части (`npm run build`) и бэкенда (`go build`).
