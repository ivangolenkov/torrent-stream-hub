# Phase 23: WebGUI UX Improvements (Bottom Panel & Piece Progress) - REPORT

## Что было сделано
1. **Модели (Go):** В `models/torrent.go` добавлены поля `PieceLength` и `NumPieces` в `models.Torrent`, а также поле `Offset` в `models.File`. Эти данные необходимы для вычисления того, какие кусочки торрента принадлежат какому файлу.
2. **API Endpoint (Go):** Создан новый эндпоинт `GET /api/v1/torrent/{hash}/pieces`, который возвращает строку-карту состояний всех кусочков торрента (0 - отсутствует, 1 - в процессе, 2 - скачан) в виде обычного текста (text/plain) для минимизации оверхеда при парсинге.
3. **Компонент PieceProgressBar (Vue):** Написан новый компонент на базе `<canvas>` для высокопроизводительной отрисовки прогресса. Компонент вычисляет, какие части глобального `piecesString` принадлежат переданному файлу (исходя из `offset` и `size`), и рисует классическую "полоску со штрих-кодом", где синим отмечены скачанные части, а зеленым - те, что в процессе загрузки. 
4. **Компонент BottomPanel (Vue):** Выезжающая шторка `TorrentInspector.vue` была удалена и заменена на нижнюю панель (Bottom Panel).
   - Реализована функция перетаскивания (Resizer) за верхнюю границу панели, чтобы пользователь мог изменять её высоту.
   - Панель разбита на вкладки "General" (для общей статистики пиров и трекеров) и "Files" (список файлов).
5. **Интерфейс списка файлов:** Перенесен в Bottom Panel. Стандартный прогресс-бар заменен на компонент `PieceProgressBar`. Добавлен столбец с отображением процента загрузки в виде точного текстового значения (например, 45.2%).
6. **Очистка:** Компонент `TorrentInspector.vue` был удален, роуты и зависимости обновлены.

## Примечания
- Использование `requestAnimationFrame` при изменении размеров контейнера через `ResizeObserver` предотвращает ошибку "ResizeObserver loop limit exceeded" в браузере при ресайзе нижней панели.
- Использована строка символов (Rune builder в Go) для возврата `pieces` статуса. Это гораздо компактнее и быстрее, чем передавать массив JSON. 
- Фронтенд периодически опрашивает `/pieces` каждые 2 секунды только когда открыта вкладка "Files" и выбран активный торрент, чтобы не создавать лишней нагрузки в фоновом режиме.
