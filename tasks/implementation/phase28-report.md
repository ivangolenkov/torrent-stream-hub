# Phase 28: Управление приоритетами файлов и раздач (Отчет)

## Что было сделано

В рамках этой фазы была реализована функция управления приоритетом скачивания файлов (как индивидуально для каждого файла, так и массово для всех файлов раздачи). 

**Изменения на Backend:**
1. **Repository (`internal/repository/torrent_repo.go`)**: Добавлены методы `UpdateFilePriority` и `UpdateTorrentFilesPriority`, которые сохраняют установленный приоритет в БД (таблица `files`).
2. **Engine (`internal/engine/engine.go`)**: Добавлены методы `SetFilePriority` и `SetTorrentFilesPriority`. Данные методы ищут соответствующий файл или перебирают все файлы через интерфейсы `anacrolix/torrent` и вызывают библиотечный метод `f.SetPriority()`.
3. **UseCase (`internal/usecase/torrent_uc.go`)**: Реализованы методы-обертки `SetFilePriority` и `SetTorrentFilesPriority`, которые сначала пытаются применить изменения в движке, а затем обязательно сохраняют изменения в репозиторий.
4. **API (`internal/delivery/http/api/handlers.go`, `router.go`)**: Добавлены два REST эндпоинта (с методом `PUT`), которые принимают JSON с желаемым уровнем приоритета:
    - `PUT /api/v1/torrent/{hash}/priority`
    - `PUT /api/v1/torrent/{hash}/file/{index}/priority`

**Изменения на Frontend:**
1. **API Client (`web/src/api/client.ts`)**: Добавлены функции `setTorrentPriority` и `setFilePriority`.
2. **Store (`web/src/stores/torrentStore.ts`)**: Добавлены Actions `setTorrentPriority` и `setFilePriority`. После отправки запроса на бэкенд, стор обновляет списки файлов для немедленного отображения изменений.
3. **UI Компоненты (`web/src/components/BottomPanel.vue`)**: Во вкладку "Files" добавлена новая колонка "Priority" с выпадающим списком (`select`) для каждого файла. Доступны состояния: Skip (не скачивать), Normal (обычный) и High (высокий). Для мобильной версии выпадающий список добавлен под названием файла.
4. **UI Компоненты (`web/src/components/TorrentTable.vue`)**: В основную таблицу добавлена колонка "Priority" с таким же выпадающим списком. При выборе опции этот компонент вызывает массовое обновление для всего торрента и возвращает `select` в состояние placeholder'а ("Set..."). Добавлена поддержка в мобильном Card View.
