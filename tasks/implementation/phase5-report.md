# Отчет по реализации: Фаза 5 (Web GUI)

## Что было сделано:
1. **Инициализация фронтенда**: Установлены и настроены библиотеки `pinia` (стейт-менеджмент), `vue-router` (роутинг), `@tailwindcss/vite` (Tailwind CSS 4.0) и `@heroicons/vue` (иконки).
2. **API и SSE клиент (`src/api/client.ts`)**:
   - Написан слой для отправки REST-запросов (`getTorrents`, `addTorrent`, `action`, `getFiles`).
   - Реализована функция `setupSSE` для инициализации `EventSource`, которая слушает `/api/v1/events` и парсит входящий поток данных для UI.
3. **Хранилище (`src/stores/torrentStore.ts`)**:
   - Создан Pinia store (`useTorrentStore`), который управляет списком торрентов, активным выбранным хешем (для инспектора файлов) и состоянием SSE-подключения.
4. **Компоненты интерфейса**:
   - **Главный лейаут (`MainLayout.vue`) и Страница (`Home.vue`)**: Адаптивная структура на Grid/Flex, объединяющая дашборд, таблицу и боковой инспектор.
   - **`DashboardStats.vue`**: Вычисляет и отображает суммарную статистику (скорость скачивания/отдачи, кол-во активных раздач) на лету (computed).
   - **`TorrentTable.vue`**: Красивая таблица с бейджами статусов (включая кастомные `DiskFull`, `MissingFiles`), прогресс-барами и кнопками управления жизненным циклом (Pause, Resume, Delete).
   - **`TorrentInspector.vue`**: Правая панель, отображающая список файлов внутри выбранного торрента. В ней присутствует вычисление загруженных процентов для каждого файла и кнопка Play для медиафайлов.
   - **`AddTorrentModal.vue`**: Модальное окно для добавления magnet-ссылки. Подготовлена визуальная зона для Drag-and-Drop (пока работает как UI-заглушка для файлов и реально добавляет по тексту magnet).
   - **`VideoPlayer.vue`**: Полноэкранное модальное окно с тегом `<video autoplay controls>`. Плеер напрямую обращается к эндпоинту `/stream?hash=...&index=...`, позволяя браузеру запрашивать куски видео через `Range`-заголовки.
5. **Тестирование сборки**: Запущена команда `npm run build` (включая проверку `vue-tsc`). Исправлены мелкие неиспользуемые импорты, сборка `dist/` завершается успешно без ошибок типов. В `vite.config.ts` добавлен прокси для локальной разработки.

## Дополнение: исправление модального окна добавления
1. Добавление magnet-ссылки больше не зависит только от SSE: после успешного ответа API модальное окно принудительно обновляет список торрентов через `store.loadTorrents()`.
2. Drag-and-Drop и file input теперь реально обрабатывают `.torrent` файл и отправляют его на backend через `multipart/form-data` в `/torrent/upload`.
3. Для ввода info hash добавлена нормализация в magnet URI на фронтенде.
4. Backend добавления magnet больше не блокирует HTTP-запрос ожиданием metadata (`GotInfo`), поэтому кнопка `Add Torrent` не зависает в состоянии `Adding...` при медленном получении метаданных.
5. После получения metadata engine теперь сохраняет список файлов в модель торрента, чтобы инспектор мог показать дерево файлов и кнопки Play.
6. Проверки после исправления: `npm run build`, `go test ./...`, `CGO_ENABLED=0 go build ./cmd/hub`.

## Дополнение: доработка UX модального окна
1. Модальное окно разделено на два явных режима: `Magnet / hash` и `.torrent file`.
2. Для `.torrent` добавлена отдельная видимая кнопка `Choose file`, drag-and-drop зона и отдельное действие `Upload .torrent`.
3. Для magnet/info hash добавлена отдельная кнопка `Add magnet` и пояснение про авто-нормализацию BTIH hash.
4. Исправлено закрытие окна после успешного добавления: форма сбрасывается, список торрентов принудительно обновляется через `store.loadTorrents()`, затем модалка закрывается.
5. Исправлена обработка пустой медиатеки: API-клиент и репозиторий больше не возвращают `null` вместо пустого массива, чтобы UI не падал до первого торрента.
6. В dev proxy Vite добавлен маршрут `/torrent`, чтобы upload `.torrent` работал в режиме разработки.
7. Исправлен stacking модального окна: модалка теперь рендерится через `Teleport` в `body`, overlay и карточка разведены по отдельным fixed-слоям, а `pointer-events` не позволяют overlay перехватывать клики внутри окна.
8. API-клиент frontend теперь читает JSON-ошибки backend и показывает поле `error`, а после pause/resume/delete store принудительно обновляет список торрентов.
