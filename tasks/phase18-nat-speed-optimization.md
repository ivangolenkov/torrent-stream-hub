# Phase 18: NAT Speed Optimization

## Описание
В условиях отсутствия проброшенных портов (NAT) и особенностей парсинга `uTP holepunch` пакетов в `anacrolix/torrent` наблюдается агрессивный сброс потенциально хороших пиров, что приводит к снижению общей скорости загрузки.
Цель этой фазы — оптимизировать параметры движка для максимизации скорости в NAT-окружении за счет переключения на TCP, использования агрессивных лимитов дозвона и увеличения времени на установку рукопожатий.

## Задачи
1. **Отключение uTP по умолчанию:**
   - Изменить дефолтное значение `BTDisableUTP` на `true` в `internal/config/config.go` (пока не будет исправлен баг в `anacrolix/torrent`).
2. **Адаптация профиля соединений:**
   - Повысить лимиты профиля `balanced` (по умолчанию) до уровня `aggressive` для более активного поиска пиров (`BTEstablishedConns=200`, `BTHalfOpenConns=200`, `BTTotalHalfOpen=2000`, `BTDialRateLimit=200`).
3. **Тонкая настройка таймаутов в Engine:**
   - Оставить короткие таймауты на установку сокета (`NominalDialTimeout=5s`, `MinDialTimeout=2s`).
   - Увеличить `HandshakesTimeout` до `20s`, чтобы пиры успевали отправить рукопожатие после установления соединения.
4. **Оптимизация хэширования:**
   - Установить `PieceHashersPerTorrent = 4` в `ClientConfig` (вместо дефолтных 2), чтобы избежать "узкого горлышка" процессора при возросшей скорости.
5. **Обновление тестов:**
   - Обновить `config_test.go` с учетом новых дефолтных значений.

## Definition of Done (DoD)
- [ ] Дефолтное значение `BTDisableUTP` изменено на `true`.
- [ ] Профиль `balanced` использует агрессивные лимиты соединений.
- [ ] В `engine.go` увеличен `HandshakesTimeout` до 20 секунд и `PieceHashersPerTorrent` установлен в 4.
- [ ] Тесты (`go test ./...`) успешно проходят.
- [ ] Отчет о реализации сохранен в `tasks/implementation/phase18-report.md`.
