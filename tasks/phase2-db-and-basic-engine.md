# Фаза 2: База данных и базовый торрент-движок (DB & Basic Engine)

Реализация уровня хранения данных и базовой обертки над `anacrolix/torrent` для классического скачивания.

## Задачи

1. **Интеграция SQLite:**
   - Настроить подключение к SQLite с обязательным `PRAGMA journal_mode=WAL;` и пулом соединений (`internal/repository/sqlite.go`).
   - Создать миграции для таблиц торрентов, файлов и истории просмотров.
2. **Инициализация торрент-клиента (`anacrolix/torrent`):**
   - Настроить клиент: указать директорию загрузки, лимиты скоростей, порты (из конфига).
   - *Важно:* Ограничить использование `mmap` и настроить предварительную аллокацию (fallocate) согласно п. 6.1.
3. **Базовое управление раздачами (Фон):**
   - Реализовать добавление торрентов (magnet и .torrent файлы).
   - Запустить фоновое скачивание по алгоритму "rarest-first".
   - Реализовать базовый жизненный цикл: Пауза, Возобновление, Удаление файлов.
4. **Контроль ресурсов (Disk Pressure & Limits):**
   - Реализовать проверку свободного места на диске. При падении ниже `HUB_MIN_FREE_SPACE_GB` переводить активные загрузки в `DiskFull`, а новые в `Queued`.
   - Ограничить количество одновременных загрузок через `HUB_MAX_ACTIVE_DOWNLOADS`.

## Definition of Done (DoD)

- [ ] Подключена и настроена база данных SQLite (WAL режим включен).
- [ ] Написаны и работают базовые миграции для хранения метаданных.
- [ ] Инициализирован движок `anacrolix/torrent` с настройками из конфигурации (включая лимиты `mmap`).
- [ ] Реализовано фоновое скачивание торрентов, постановка на паузу и удаление.
- [ ] Написан механизм проверки свободного места на диске, корректно меняющий статусы загрузок на `DiskFull` и `Queued`.
