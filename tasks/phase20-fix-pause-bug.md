# Phase 20: Fix Torrent Pause Functionality

## Описание
В текущей реализации метод `Pause()` в `internal/engine/engine.go` не останавливает фактическое скачивание торрента на уровне движка `anacrolix/torrent`. 
Торрент продолжает качаться в фоне, игнорируя статус `Paused`.

## Причина
При старте скачивания мы вызываем метод `mt.t.DownloadAll()`, который проходит по всем кусочкам (pieces) торрента и жестко задает им приоритет `PiecePriorityNormal`.
При попытке поставить торрент на паузу, текущий код делает следующее:
```go
for _, f := range mt.t.Files() {
    f.SetPriority(torrent.PiecePriorityNone)
}
```
Он сбрасывает приоритеты только на уровне файлов. Однако внутренний алгоритм `anacrolix/torrent` высчитывает эффективный приоритет как максимум между приоритетом файла и приоритетом самого кусочка. Так как кусочки сохраняют статус `Normal`, скачивание продолжается.

## Задача
Необходимо корректно сбрасывать приоритет самих кусочков (pieces) при вызове паузы.
