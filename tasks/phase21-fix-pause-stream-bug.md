# Phase 21: Fix Pause State Override by Stream Manager

## Описание
Был обнаружен баг: торрент, поставленный на паузу, продолжает или начинает скачиваться после того, как плеер завершает стриминг (отключается).

## Причина
В `internal/engine/stream.go` при завершении стриминга (в `setSequentialMode` когда `enable=false`), движок слепо сбрасывает приоритет всех файлов в `torrent.PiecePriorityNormal`. Если торрент на тот момент был поставлен пользователем на паузу (статус `Paused`), установка `PiecePriorityNormal` заставляет `anacrolix/torrent` немедленно возобновить скачивание, ломая семантику паузы. 

## Задача
Внести проверку статуса торрента в `setSequentialMode`. При выключении Sequential Mode (`enable == false`) файлы должны получать приоритет `Normal` только если торрент находится в активном состоянии (`Downloading`, `Seeding`, `Queued`). Если торрент в состоянии `Paused` или `Error`, приоритет файлов должен остаться `None`, или, вернее, нужно пропустить установку `Normal`. Более надежный вариант — перед изменением приоритета проверять `mt.state`.

## Definition of Done (DoD)
- [ ] В `internal/engine/stream.go` (метод `setSequentialMode`) добавлена проверка статуса `mt.state`.
- [ ] Если `enable == false` и статус торрента `Paused` (и другие неактивные статусы), мы восстанавливаем приоритеты как `None`, либо просто игнорируем восстановление. (Или лучше устанавливаем `None`, если он на паузе).

- [ ] Все тесты успешно проходят (`go test ./...`).
