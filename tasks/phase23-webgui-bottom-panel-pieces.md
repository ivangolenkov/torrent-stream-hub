# Phase 23: WebGUI Redesign (Bottom Panel) & Piece Progress

## Описание
Замена выезжающей боковой панели (Torrent Inspector) на классическую нижнюю панель (Bottom Panel) с вкладками, аналогичную популярным клиентам (qBittorrent, uTorrent). Добавление визуального отображения карты кусочков (Piece map) для каждого файла с использованием HTML5 `<canvas>` для высокой производительности, а также текстового отображения процента скачивания файлов.

## Задачи

### 1. Бэкенд (Go)
- [ ] Добавить новое поле `Offset` (смещение файла в байтах от начала торрента) в модель `models.File`.
- [ ] Расширить `models.Torrent` или создать новый эндпоинт для передачи информации о кусочках (`PieceLength`, общее количество кусочков, и битовая маска или строка с состояниями всех кусочков торрента). 
- [ ] Добавить быстрый эндпоинт `GET /api/v1/torrent/{hash}/pieces`, возвращающий состояние скачивания кусочков (например, `PieceStateRuns` или массив статусов) для отрисовки графиков. Возвращаемые данные должны быть максимально легковесными (например, строка "2221000", где цифра означает статус).

### 2. Фронтенд: Piece Map (Vue)
- [ ] Создать компонент `PieceProgressBar.vue` на базе HTML5 `<canvas>`.
- [ ] Компонент должен принимать данные о размере файла, `offset` внутри торрента, размере `piece` и текущем массиве статусов кусочков (с нового эндпоинта).
- [ ] Нарисовать линию (как в классических клиентах), отображающую, какие части файла скачаны (например, синий цвет), какие качаются (зеленый), а какие отсутствуют (серый).

### 3. Фронтенд: Bottom Panel (Vue)
- [ ] Заменить выезжающую боковую панель (`TorrentInspector.vue`) на закрепленную снизу (Bottom Panel) на главной странице (`Home.vue`).
- [ ] Реализовать возможность изменения высоты панели с помощью мыши (Resizer).
- [ ] Добавить структуру вкладок (Tabs) в нижнюю панель:
  - **General**: Статистика пиров, информация о трекерах (из старого инспектора).
  - **Files**: Список файлов.

### 4. Фронтенд: Обновление списка файлов
- [ ] Переместить логику отображения файлов в новую вкладку **Files**.
- [ ] Рядом с каждым файлом вывести точный текстовый процент скачивания без скобок (например, `45.2%`).
- [ ] Заменить старый прогресс-бар (обычную полоску) на новый компонент `PieceProgressBar.vue` с картой кусочков.
- [ ] Кнопка Play (стриминг) для медиафайлов должна остаться.

## Definition of Done (DoD)
- [ ] Модели в `models/torrent.go` обновлены.
- [ ] Добавлен и протестирован эндпоинт `GET /torrent/{hash}/pieces`.
- [ ] Вместо бокового Drawer реализована нижняя панель с вкладками (General, Files).
- [ ] Высота нижней панели может быть изменена мышью пользователем (настраиваемый сплиттер).
- [ ] Файлы отображают текстовый процент готовности и `<canvas>` карту кусочков.
- [ ] Приложение (frontend) успешно собирается `npm run build`.
- [ ] Сервер успешно собирается и тесты проходят `go test ./...`.
