# Phase 25: WebGUI HTTP Downloads

## Описание
Добавить в Web GUI возможность скачивания полностью загруженных файлов и раздач по HTTP.

Скачивание отдельного файла должно поддерживать восстановление после паузы за счет HTTP `Range`-запросов. Скачивание всей раздачи должно работать только для полностью скачанной раздачи: если в ней один файл, скачивается именно этот файл; если файлов несколько, сервер формирует ZIP-архив на лету со всеми файлами и папками. Для потокового ZIP восстановление загрузки не требуется.

Файлы для HTTP-скачивания должны читаться напрямую с диска, а не через torrent reader.

## Задачи

### 1. Backend API (Go)
- [ ] Добавить эндпоинты `GET /api/v1/torrent/{hash}/download` и `HEAD /api/v1/torrent/{hash}/download` для скачивания всей раздачи.
- [ ] Добавить эндпоинты `GET /api/v1/torrent/{hash}/file/{index}/download` и `HEAD /api/v1/torrent/{hash}/file/{index}/download` для скачивания отдельного файла.
- [ ] Для незавершенных файлов и раздач возвращать `409 Conflict`.
- [ ] Для отсутствующего торрента возвращать `404 Not Found`.
- [ ] Для некорректного индекса файла возвращать `400 Bad Request`.
- [ ] Для торрента без metadata возвращать `409 Conflict`.

### 2. Безопасное чтение файлов с диска
- [ ] Реализовать переиспользуемую логику безопасного разрешения пути файла внутри `downloadDir`.
- [ ] Учитывать текущие варианты расположения файлов: `<downloadDir>/<file.path>` и `<downloadDir>/<torrent.name>/<file.path>`.
- [ ] Запретить абсолютные пути, `..` и любые пути, выходящие за пределы `downloadDir`.
- [ ] Для скачивания выбирать первый существующий regular file с ожидаемым размером.
- [ ] Если файл помечен как полностью скачанный, но на диске отсутствует или имеет неправильный размер, возвращать `409 Conflict` как признак рассинхронизации состояния.

### 3. Скачивание отдельного файла
- [ ] Проверять, что `file.downloaded >= file.size` и файл найден на диске.
- [ ] Отдавать файл через `http.ServeContent` поверх `os.File`, чтобы сохранить поддержку `Range`, `HEAD`, `Content-Length` и resume.
- [ ] Выставлять `Content-Disposition: attachment` с корректным UTF-8 именем файла.
- [ ] Выставлять `Accept-Ranges: bytes` и корректный `Content-Type`.

### 4. Скачивание всей раздачи
- [ ] Проверять, что все файлы раздачи полностью скачаны.
- [ ] Если в раздаче один файл, отдавать его тем же кодом, что и скачивание отдельного файла.
- [ ] Если файлов несколько, формировать ZIP-архив на лету через `archive/zip`.
- [ ] Для каждого ZIP entry использовать нормализованный относительный путь файла с `/` в качестве разделителя.
- [ ] Сохранять структуру папок внутри ZIP.
- [ ] Для потокового ZIP выставлять `Content-Type: application/zip` и `Content-Disposition: attachment`.
- [ ] Не выставлять `Content-Length` для потокового ZIP.
- [ ] При `Range`-запросе к потоковому ZIP возвращать `416 Range Not Satisfiable`.
- [ ] Корректно закрывать открытые файлы и ZIP writer при отмене запроса клиентом.

### 5. CORS и заголовки
- [ ] Добавить `Content-Disposition` в `ExposedHeaders` CORS-настроек.
- [ ] Убедиться, что существующие `Accept-Ranges`, `Content-Length`, `Content-Range` и `Content-Type` остаются доступными клиентам.

### 6. Frontend Web GUI (Vue)
- [ ] Добавить helper-методы URL в `web/src/api/client.ts`: `torrentDownloadUrl(hash)` и `fileDownloadUrl(hash, index)`.
- [ ] В `TorrentTable.vue` добавить пункт меню `Download` для раздачи.
- [ ] Показывать кнопку скачивания раздачи только если `torrent.size > 0` и `torrent.downloaded >= torrent.size`.
- [ ] В `BottomPanel.vue` на вкладке `Files` добавить кнопку скачивания напротив файла.
- [ ] Показывать кнопку скачивания файла только если `file.size > 0` и `file.downloaded >= file.size`.
- [ ] Запускать скачивание через обычную ссылку или `window.location.href`, не через `fetch`, чтобы браузер мог управлять загрузкой и resume для одиночных файлов.
- [ ] Сохранить существующую кнопку `Play` для медиафайлов.

### 7. Тестирование
- [ ] Добавить backend-тесты для успешного скачивания завершенного файла с диска.
- [ ] Добавить backend-тест на `Range` для отдельного файла с ответом `206 Partial Content`.
- [ ] Добавить backend-тест на `409 Conflict` для незавершенного файла.
- [ ] Добавить backend-тест на однофайловую раздачу: скачивается сам файл, а не ZIP.
- [ ] Добавить backend-тест на многофайловую раздачу: возвращается ZIP со структурой папок и корректным содержимым.
- [ ] Добавить backend-тест на `416 Range Not Satisfiable` для `Range`-запроса к потоковому ZIP.
- [ ] Добавить backend-тест на отклонение unsafe path.
- [ ] Проверить backend командой `go test ./...`.
- [ ] Проверить frontend сборкой `npm run build` в директории `web`.
