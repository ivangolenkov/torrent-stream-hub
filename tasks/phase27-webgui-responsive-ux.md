# Phase 27: Web GUI Responsive UX & Polish

## Описание
Улучшение пользовательского опыта (UX) и адаптивности Web GUI на мобильных устройствах. Переход от классических таблиц к карточкам на узких экранах, улучшение поведения нижней панели инспектора и добавление информативных "пустых состояний" (Empty States).

## Задачи

### 1. Адаптивность таблиц (Responsive Cards)
- **Файл:** `web/src/components/TorrentTable.vue`
- Изменить структуру списка торрентов: на десктопах (`>= md`) оставить классическую таблицу.
- На мобильных (`< md`) скрыть `<thead>` и преобразовать каждую строку `<tr>` в flex-карточку.
- **Файл:** `web/src/components/BottomPanel.vue` (Вкладка Files)
- Применить аналогичный паттерн к списку файлов торрента. На мобильных экранах файлы должны отображаться в виде карточек.

### 2. Мобильный инспектор (Bottom Sheet)
- **Файл:** `web/src/components/BottomPanel.vue`
- На мобильных экранах (`< md`) отключить ручной ресайз панели (убрать курсор-ресайзер и логику пересчета высоты по мыши).
- Зафиксировать высоту панели на мобильных устройствах (например, `85vh`), чтобы она выглядела как всплывающее снизу окно (Bottom Sheet). На десктопе оставить как было (с ресайзом).
- Добавить плавную CSS-анимацию появления панели.

### 3. Адаптивное меню навигации
- **Файл:** `web/src/layouts/MainLayout.vue`
- На очень узких экранах скрывать текстовое название `TorrentStreamHub` (оставляя только логотип `PlayIcon`).
- Реализовать скрытие навигационных ссылок в выпадающее "Гамбургер-меню" на мобильных экранах (`< sm`), чтобы предотвратить перенос меню на новую строку и поломку шапки.

### 4. Дизайн "Пустых состояний" (Empty States)
- **Файл:** `web/src/components/TorrentTable.vue`
- Если список торрентов пуст, вместо текста "No torrents added yet" отрендерить красивый блок с иконкой, поясняющим текстом и акцентной кнопкой "Add Torrent", открывающей модальное окно добавления.
- **Файл:** `web/src/components/BottomPanel.vue` (Вкладка Files)
- Улучшить визуальное представление состояния "Загрузка файлов...", пока метаданные еще не получены.

## Definition of Done (DoD)
- [ ] На мобильном устройстве (ширина экрана < 768px) список торрентов отображается в виде удобных карточек, горизонтальный скролл отсутствует.
- [ ] На мобильном устройстве список файлов внутри торрента отображается в виде карточек, горизонтальный скролл отсутствует.
- [ ] На мобильном устройстве инспектор (BottomPanel) открывается как "Bottom Sheet" на большую часть экрана, ручной ресайз отключен.
- [ ] На узких экранах шапка сайта не ломается (меню скрывается под гамбургер).
- [ ] При пустом списке торрентов отображается красивый блок Empty State с кнопкой добавления торрента.
- [ ] Все изменения корректно выглядят и не ломают отображение на десктопных разрешениях (ширина экрана >= 768px).
- [ ] Проект успешно собирается (`npm run build` внутри `web/`).
