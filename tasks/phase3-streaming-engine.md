# Фаза 3: Ядро стриминга и умная маршрутизация (Streaming Engine)

Самая критическая часть проекта — обеспечение возможности потокового просмотра до полного скачивания файла.

## Задачи

1. **Управление потоками (Reference Counting):**
   - Реализовать счетчик `activeStreams` для каждого файла раздачи с привязкой к HTTP-контексту (`context.Context`).
2. **Переключение режимов (QoS):**
   - Если `activeStreams > 0`, переключать загрузку файла в режим `Sequential` (приоритет первого и последнего кусков для метаданных).
   - После скачивания 100% стримингового файла, отключать `Sequential` и снимать ограничения QoS для докачивания остатков в фоне (п. 6.2.5).
3. **Debounce-механизм перемотки:**
   - Реализовать таймер задержки (5-10 сек) при обрыве соединения/смене `Range`. Удерживать режим `Sequential` до истечения таймера, чтобы избежать скачков приоритетов при перемотке плеером.
4. **Эмуляция кэша (`/cache`):**
   - Написать алгоритм "скользящего окна": проверка `bitfield` для подсчета непрерывно скачанных кусков вперед от текущего индекса воспроизведения.
   - Реализовать искусственный лимит возвращаемого значения `HUB_STREAM_CACHE_SIZE` (п. 6.3).

## Definition of Done (DoD)

- [ ] Механизм счетчика `activeStreams` корректно увеличивается/уменьшается при подключении/отключении клиентов.
- [ ] При `activeStreams > 0` файл скачивается последовательно (Sequential).
- [ ] Работает таймер задержки (Debounce), удерживающий режим Sequential при обрывах соединения.
- [ ] По достижении 100% загрузки файла, режим Sequential отключается.
- [ ] Написана логика эмуляции кэша, возвращающая объем скачанных данных вперед с учетом лимита `HUB_STREAM_CACHE_SIZE`.
