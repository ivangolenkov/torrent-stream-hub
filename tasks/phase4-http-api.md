# Фаза 4: Разработка HTTP API (REST & TorrServer Layer)

Создание интерфейсов для взаимодействия с Web GUI и внешними ТВ-клиентами.

## Задачи

1. **REST API для Web GUI (`/api/v1/...`):**
   - Эндпоинты для получения списка, добавления, управления жизненным циклом и получения дерева файлов.
2. **Server-Sent Events (SSE):**
   - Создать эндпоинт `/api/v1/events` для потоковой передачи изменений статусов торрентов в реальном времени. Настроить воркер, опрашивающий движок и рассылающий события.
3. **Слой совместимости TorrServer:**
   - Реализовать эндпоинты: `/echo`, `/torrents` (добавление/список в формате TorrServer).
   - Создать ядро стриминга: эндпоинты `/stream` и `/play/{hash}/{id}` с поддержкой HTTP `Range` запросов (206 Partial Content).
   - Заглушки и вспомогательные эндпоинты: `/settings`, `/viewed`, `/playlist` (генерация M3U).
4. **Настройка сети:**
   - Настроить CORS: разрешить `*` для всех эндпоинтов (требование для интеграции со Smart TV).

## Definition of Done (DoD)

- [ ] Реализованы все эндпоинты REST API для Web GUI согласно спецификации OpenAPI.
- [ ] Работает эндпоинт SSE (`/api/v1/events`), транслирующий актуальное состояние торрентов.
- [ ] Реализованы эндпоинты совместимости с TorrServer (`/echo`, `/torrents`, `/settings`, `/viewed`, `/playlist`).
- [ ] Реализован эндпоинт потоковой передачи видео (`/stream` и `/play`) с корректной обработкой `Range` запросов (206 Partial Content).
- [ ] CORS настроен на разрешение запросов с любых источников (`*`).
