# Фаза 5: Разработка Web GUI (Frontend)

Создание пользовательского интерфейса для управления сервером.

## Задачи

1. **Базовая структура и State Management:**
   - Настроить Pinia для хранения списка торрентов.
   - Подключить SSE-клиент для автоматического обновления стейта в браузере.
2. **Компоненты интерфейса:**
   - **Дашборд:** Вывод общей статистики и графиков.
   - **Таблица раздач (Грид):** Отображение прогресса, статусов (с бейджами вроде "Стримится", "DiskFull"), скоростей.
   - **Инспектор (Детали):** Вкладка с деревом файлов, выбор приоритетов скачивания.
3. **Управление и плеер:**
   - Модальное окно добавления торрента с поддержкой Drag-and-Drop.
   - Кнопка "Play" напротив файлов для запуска видеопотока прямо в браузере через стандартный тег `<video>` (используя эндпоинт `/stream`).

## Definition of Done (DoD)

- [ ] Настроено хранилище Pinia для состояния торрентов.
- [ ] Реализовано подключение к SSE и автоматическое обновление интерфейса без перезагрузки страницы.
- [ ] Реализован Дашборд с базовой статистикой.
- [ ] Реализована таблица торрентов с отображением всех необходимых данных (прогресс, скорость, статус).
- [ ] Реализован Инспектор для просмотра дерева файлов и управления их приоритетами.
- [ ] Работает добавление торрентов через UI (включая Drag-and-Drop).
- [ ] Видеофайлы можно запустить прямо в браузере через встроенный плеер.
