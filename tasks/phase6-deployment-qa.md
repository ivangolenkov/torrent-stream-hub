# Фаза 6: Интеграция, деплой и тестирование (Deployment & QA)

Объединение фронтенда и бэкенда в единый продукт.

## Задачи

1. **Встраивание Frontend (Go Embed):**
   - Настроить Vite для сборки статики в директорию `web/dist`.
   - Использовать `//go:embed web/dist` в Go-коде для раздачи статики веб-сервером бэкенда.
2. **Docker-контейнеризация:**
   - Написать Multi-stage `Dockerfile`: стадия сборки Node.js (Vue), стадия сборки Go (бинарник), и финальный легковесный образ (например, Alpine).
   - Создать `docker-compose.yml` согласно ТЗ с пробросом портов (8080, 50007 TCP/UDP) и вольюмов (`/downloads`, `/config`).
3. **Интеграционное тестирование:**
   - Проверить сценарий "ТВ-клиент" через плееры (Lampa/Num или VLC) — перемотка, стабильность стрима.
   - Проверить сценарий нехватки места на диске (`DiskFull`).
   - Проверить работу SSE в веб-интерфейсе.

## Definition of Done (DoD)

- [ ] Фронтенд успешно собирается и встраивается в бинарный файл бэкенда через `go:embed`.
- [ ] Создан рабочий `Dockerfile`, собирающий единый образ приложения.
- [ ] Создан `docker-compose.yml`, позволяющий запустить приложение со всеми нужными настройками.
- [ ] Приложение успешно запускается в Docker.
- [ ] Ручное тестирование (VLC/браузер) подтверждает стабильную потоковую передачу.
